| 單頁式導覽：scroll-spy 底線指示 + 錨點捲動 | `src/components/site-header.tsx`、`src/app/[locale]/page.tsx`、`src/app/[locale]/works/page.tsx`、`src/app/[locale]/about/page.tsx`、`src/app/[locale]/works/[slug]/page.tsx` | 原本首頁/作品/關於是三個獨立路由，改成合併進首頁的三個區塊（`id="home"/"works"/"about"`），`/works`、`/about` 改為對 `/#works`、`/#about` 的 redirect stub（保留舊連結相容）。`SiteHeader` 改成 client component：用 `window.addEventListener('scroll', ...)` 走訪三個 section 比較 `offsetTop` 決定目前分頁（比 `IntersectionObserver` + 負 bottom margin 更穩健——後者在最後一個 section 比視窗矮時，永遠無法達到「捲動進視窗 55%」的門檻，導致「關於」分頁永遠不會被判定為 active；改用純捲動位置比較後，額外加上「已捲到頁面最底部」的特判，確保短 section 也能正確 active）。分頁底線用 Framer Motion `layoutId` 做滑動動畫；點擊導覽列在首頁上時呼叫 `useLenis()` 的 `lenis.scrollTo()` 平滑捲動並用 `history.replaceState` 更新 hash（不觸發整頁導航），在非首頁（如作品詳情頁）點擊則走一般 `Link` 導航回首頁對應錨點。Header 改 `sticky top-0`，讓使用者捲動時能持續看到分頁指示器變化。 |
| 修正 locale 路由未真正靜態化 | `src/app/[locale]/layout.tsx`、`page.tsx`、`about/page.tsx`、`works/page.tsx`、`works/[slug]/page.tsx` | `npm run build` 檢查發現所有 `[locale]` 路由標示為 `ƒ (Dynamic)` 而非 `○/● (Static)`，即使已有 `generateStaticParams`：原因是 next-intl 的 `requestLocale` 來自 request-time 偵測，若沒有在每個 page/layout 呼叫 `setRequestLocale(locale)` 明確聲明，Next.js 無法在 build time 靜態渲染。修正後改用 `getTranslations`（`next-intl/server`）取代 async component 內不能用的 `useTranslations`/`useLocale` hook 版本；重新 build 後全部路由變成 `● (SSG)`，代表未來部署會直接吃 Vercel CDN 快取而非每次 request 都跑 server render。 |
| Vercel 靜態資源快取設定 | `vercel.json` | 新增 `vercel.json`，對 `/static/*`（Velite 產出、`[name]-[hash:6].[ext]` content-hash 檔名）設定 `Cache-Control: public, max-age=31536000, immutable`，讓 CDN/瀏覽器長期快取而不需 revalidate；因檔名本身含內容雜湊，內容變更會自動產生新檔名，不會有快取到舊內容的風險。 |

| 修正過頂回彈定格過久 + 全域彈跳規則吃掉箭頭淡入動畫 | `src/components/overscroll-bounce.tsx`、`src/app/globals.css` | 使用者回報兩個問題：①從頂部往下拉時會「定格很久才彈回去」，沒有自然彈跳感；②電腦版照片輪播 hover 到圖片時，左右切換箭頭不見了。用 preview 工具實測找到根因：①Chrome/Edge 觸控板放開手指後仍會送出長達 1~1.5 秒、逐漸衰減的「慣性捲動」wheel 事件（實測 31 個事件跨 1.56 秒），而原本邏輯每收到一個事件（哪怕 deltaY 只剩 0.5）就會重設 150ms 閒置計時器，導致整個慣性尾巴期間畫面都卡在拉伸狀態，直到真正完全靜止才彈回去；修法是加 `MAX_HOLD_MS=400`，用 `performance.now()` 量測手勢從第一次拉動起經過的實際時間，超過 400ms 就不再等待閒置、直接強制 `release()`，把「卡住」時間上限鎖定在 0.4 秒內；同時把彈回動畫從 `stiffness:300/damping:28`（阻尼比 ~0.81，接近臨界阻尼、幾乎不回彈）改成 Framer Motion 的 `bounce:0.35, duration:0.5` 參數化寫法，讓放開後有明顯但不誇張的回彈手感。②追查 `src/app/globals.css` 的全域 `button:not(:disabled) { transition-property: transform !important }` 規則（上一輪「全站按鈕彈跳回饋」新增），發現它會把箭頭按鈕自己的 `transition-opacity`（`photo-carousel.tsx:79,87`）連同 CTA/卡片的 `transition-shadow` 一起蓋掉，導致 hover 顯示/隱藏從平滑淡入變成生硬瞬間切換；修法是把 `transition-property` 從單一 `transform` 擴充成 `transform, opacity, box-shadow`（涵蓋目前程式碼裡所有按鈕會用到的過渡屬性），維持點擊彈跳感的同時不再犧牲各按鈕自己的 hover 過渡。**驗證方式**：用 preview 工具直接對 `window` 派送模擬慣性尾巴的 wheel 事件序列（前段 6 次強力滑動 + 後段 25 次指數衰減的微小 deltaY），實測確認舊邏輯會被最後一個事件的 150ms 閒置計時器一路拖到 1.5 秒以上；CSS 修正則直接 fetch 編譯後的 CSS chunk 內容比對 `transition-property` 實際輸出，確認從 `transform` 變成 `transform, opacity, box-shadow`。**除錯記錄**：驗證過程中發現 Turbopack dev server 在改動 `globals.css` 後，即使重啟 `npm run dev` 仍持續透過 `304 Not Modified`／舊 `Last-Modified` header 提供修改前的舊 CSS chunk（`.next` 持久化快取沒有正確失效這次的 CSS 內容變更），必須整個刪除 `.next` 目錄重新啟動才會反映最新內容；另外自動化工具用 `dispatchEvent()` 送出的合成滑鼠事件不會觸發瀏覽器原生 `:hover` 偽類，也不能用來驗證 `group-hover` 是否顯示，改用直接讀取編譯後 CSS 內容 + 邏輯推導佐證。 |
| 全站按鈕加上點擊彈跳回饋 | `src/app/globals.css`、`src/components/ui/button.tsx`、`src/components/site-header.tsx`、`src/components/intro-hero.tsx`、`src/components/work-card.tsx` | 使用者要求「任何按鈕點擊後要有彈跳感」。用純 CSS（不碰事件處理邏輯，避免重蹈本 session 稍早 Base UI 點擊互動的坑）：`globals.css` 新增 `.tap-bounce` 工具類 + 一條全域 `button:not(:disabled)` 規則，按下時縮到 `scale(0.92)`，放開後用 `cubic-bezier(0.34, 1.56, 0.64, 1)`（會先衝過 1.0 再回彈，是常見的「回彈按鈕」CSS trick）彈回原大小；因為套用範圍是全域 `button` 選擇器，Sheet 關閉鈕、`Tabs` 分類頁籤、輪播箭頭、「來點互動」佔位鈕全部自動吃到，不用逐一修改。原生 `<a>`/`Link` 樣式的「按鈕」（導覽列 4 個連結、品牌連結、手機選單連結、簡介兩顆 CTA、作品卡片整張卡）手動加上 `tap-bounce` class。共用的 `ui/button.tsx`（作品詳情頁下載/外部連結按鈕）拿掉原本的 `active:...:translate-y-px` 小位移，改用 `tap-bounce`；因為它原本的 `transition-all` 會跟自訂的 transition 打架，`.tap-bounce` 的 transition 屬性加了 `!important` 確保彈跳曲線不會被蓋掉。**驗證方式**：純 CSS 效果不牽涉 JS 事件處理，用 `getComputedStyle` 逐一確認導覽列連結、品牌連結、CTA 按鈕、漢堡選單鈕、`ui/button.tsx` 渲染出的 `<a>` 標籤，全部正確吃到 `transitionProperty: transform` + 對應的回彈曲線；`:active` 實際按下狀態的視覺效果無法用自動化工具強制觸發驗證，但因為是單純 CSS 偽類（不像先前的 Base UI 互動元件需要真實事件序列），可靠度高。 |
